> このシートは、完成したツールやスクリプトが「他の場面でも使えるか」を判断するためのシートです。
> 横展開できるものを放置するのは機会損失です。積極的に活用してください。

## ブロック1：技術的な再利用可能性

| # | チェック項目 | 判定 | 具体的な内容 |
|---|---|---|---|
| 1 | 処理の設定値（URL・ID・キーワード等）が config.json または .env に外出しされているか | ○ / △ / × | |
| 2 | 業務固有の処理（フィルタ条件・整形ルール等）が関数・クラスとして分離されているか | ○ / △ / × | |
| 3 | READMEだけで他の担当者がゼロから動かせるか | ○ / △ / × | |
| 4 | .env.exampleが整備されていて、環境変数の説明が書かれているか | ○ / △ / × | |
| 5 | 設定を変えるだけで別の入力元・出力先に対応できるか | ○ / △ / × | |
| 6 | テンプレート化（汎用フレームワークとして抽出）できる処理が含まれているか | ○ / △ / × | |

## 判断基準

| スコア | 判断 |
|---|---|
| 18点以上 | **すぐに横展開を計画する** |
| 12〜17点 | **展開先を1つ決めて着手する** |
| 6〜11点 | **テンプレート化だけ先にやっておく** |
| 5点以下 | **この案件固有。今は展開しない** |
