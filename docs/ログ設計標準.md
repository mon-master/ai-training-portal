**バージョン**: 1.0.0
**作成日**: 2026-06-13

---

## 1. ログの目的

ログは「何のために残すのか」を理解した上で設計する。

| 目的 | 具体的な用途 |
|---|---|
| **障害追跡** | エラー発生時に「何が・いつ・なぜ」失敗したかを特定する |
| **運用確認** | 処理が正常に完了したかを日常的に確認する |
| **件数照合** | 処理件数と実データを突き合わせて整合性を確認する |
| **再実行支援** | 失敗したデータだけを再処理するために失敗IDを特定する |
| **監査対応** | いつ誰が何を処理したかを後から追跡できる |

---

## 2. ログレベルの使い分け

| レベル | 用途 | 例 |
|---|---|---|
| **DEBUG** | 開発時のみ。詳細なデバッグ情報（本番では原則OFF） | 変数の中身、ループの各ステップ |
| **INFO** | 正常処理の記録。運用確認に使う | 処理開始、完了、件数サマリー |
| **WARNING** | 想定範囲内の軽微な問題。処理は継続する | データ不正でスキップ、リトライ発生 |
| **ERROR** | 処理失敗。特定データや機能が動かない | API失敗、ファイル書き込み失敗 |
| **CRITICAL** | システム停止レベルの障害 | 認証エラー、設定ファイル読み込み失敗 |

### 判断チャート

```
エラーが発生した
    ↓
処理を継続できる？
    ├─ YES → WARNING（1件スキップ等）
    └─ NO  → 全体を止める必要があるか？
                  ├─ YES → CRITICAL（認証失敗・設定不正）
                  └─ NO  → ERROR（この操作だけ失敗）
```

## 3. ログファイルの命名規則

```
logs/
├── 20260613.log         # 日次ログ（推奨）
├── 20260613_0900.log    # 時次ログ（高頻度実行の場合）
└── error_20260613.log   # エラー専用ログ（任意）
```

| 項目 | 規則 |
|---|---|
| 保存先 | プロジェクトルート直下の `logs/` |
| ファイル名 | `YYYYMMDD.log`（日次）または `YYYYMMDD_HHMM.log`（時次） |
| 文字コード | UTF-8 |
| 改行コード | LF（Unix形式） |

## 4. ログフォーマット

### 4.1 標準フォーマット

```
[YYYY-MM-DD HH:MM:SS] [LEVEL] [exec_id:{UUID}] {メッセージ}
```

## 5. 実行ID（exec_id）

| 項目 | 詳細 |
|---|---|
| **目的** | 1回の実行ログを他の実行と区別するための一意なID |
| **形式** | UUIDv4（例: `a1b2c3d4-e5f6-...`）または `YYYYMMDD_HHMMSS` |
| **生成タイミング** | プロセス起動時に1回だけ生成 |
| **使い方** | 全ログ行に付与する。障害発生時にgrepで絞り込める |

```python
import uuid
exec_id = str(uuid.uuid4())[:8]  # 短縮版（8文字）
```

## 6. ログに残す情報

### 6.1 必須の情報

| 情報 | 説明 |
|---|---|
| 実行日時 | ISO 8601形式で秒まで記録 |
| 実行ID | exec_id |
| 実行モード | 本番 / dry-run |
| 処理件数（合計） | 入力データの総数 |
| 成功件数 | 正常に処理できた件数 |
| 失敗件数 | エラーが発生した件数 |
| スキップ件数 | 意図的に除外した件数（重複・不正データ） |
| エラー内容 | エラーメッセージ・エラー種別 |
| 失敗したデータのID | 再実行に必要 |
| 実行時間 | パフォーマンス監視に使う |

### 6.2 エラー時に残す追加情報

```
[ERROR] [exec_id:a1b2c3] ID:456 処理失敗
  エラー種別: HTTPError
  HTTPステータス: 500
  エンドポイント: https://api.example.com/v1/records
  リトライ回数: 3
  エラーメッセージ: Internal Server Error
  スタックトレース:
    File "src/writer.py", line 45, in write_record
    ...
```

## 7. ログに残してはいけない情報

| NG情報 | 理由 | 代替 |
|---|---|---|
| APIキー・パスワード | ログファイルが漏洩した場合に悪用される | `***` でマスキング |
| OAuthトークン | 同上 | マスキング |
| 個人のフルネーム | 個人情報保護の観点 | IDのみ記録 |
| 電話番号・メールアドレス（フル） | 同上 | 末尾マスキング（090-****-1234） |
| クレジットカード番号 | PCI-DSS違反 | 下4桁のみ |
| 受注の全明細 | 機密情報 | 件数・合計金額のみ |

## 8. ローテーションと保存期間

| 実行頻度 | ローテーション | 保存期間 |
|---|---|---|
| 日次 | 1日1ファイル | 30日 |
| 時次以上 | 1日1ファイルに集約 | 7〜30日 |
| 月次 | 月ごとに1ファイル | 1年 |

### 自動削除（Python例）

```python
import os
from datetime import datetime, timedelta

LOG_DIR = "logs"
RETENTION_DAYS = 30

cutoff = datetime.now() - timedelta(days=RETENTION_DAYS)
for filename in os.listdir(LOG_DIR):
    filepath = os.path.join(LOG_DIR, filename)
    if os.path.getmtime(filepath) < cutoff.timestamp():
        os.remove(filepath)
```

## 9. Python実装例

```python
import logging
import os
from datetime import datetime

def setup_logger(exec_id: str) -> logging.Logger:
    log_dir = "logs"
    os.makedirs(log_dir, exist_ok=True)
    
    log_file = os.path.join(log_dir, f"{datetime.now().strftime('%Y%m%d')}.log")
    
    formatter = logging.Formatter(
        f"[%(asctime)s] [%(levelname)s] [exec_id:{exec_id}] %(message)s",
        datefmt="%Y-%m-%d %H:%M:%S"
    )
    
    logger = logging.getLogger("app")
    logger.setLevel(logging.INFO)
    
    file_handler = logging.FileHandler(log_file, encoding="utf-8")
    file_handler.setFormatter(formatter)
    
    stream_handler = logging.StreamHandler()
    stream_handler.setFormatter(formatter)
    
    logger.addHandler(file_handler)
    logger.addHandler(stream_handler)
    
    return logger
```
