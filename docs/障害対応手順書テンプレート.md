## 障害レベル定義

| レベル | 定義 | 対応目標時間 | 例 |
|---|---|---|---|
| **L1 緊急** | 研修サイトが完全に表示されない | 2時間以内に復旧 | Vercelサービス障害、デプロイ完全失敗 |

## ケース別対応手順

---

### ケース1: サイトが表示されない（L1）

#### 確認手順

```
Step 1: Vercelのステータスを確認する
  → https://www.vercel-status.com/
  → 障害情報があれば待機（Vercel側の問題）

Step 2: デプロイ状況を確認する
  → https://vercel.com/dashboard でai-training-portalのデプロイログを確認
  → ビルドエラーがある場合は「ケース2」に進む

Step 3: カスタムドメインのDNSを確認する
  → ムームーDNSで ai-training CNAME が cname.vercel-dns.com を向いているか確認
```

#### 復旧手順

**Vercel側の障害の場合:** 復旧を待つ（通知メールを確認）

**デプロイ失敗の場合:** ケース2を参照

**DNSの問題の場合:**
1. ムームードメインの管理画面でDNS設定を確認
2. `ai-training` CNAME が `cname.vercel-dns.com` を向いているか確認
3. 設定が正しければ、DNS反映を待つ（最大48時間）

---

### ケース2: Vercelのビルドが失敗する（L2〜L3）

#### 確認手順

```powershell
# 1. Vercel管理画面でエラーログを確認
#    → https://vercel.com → ai-training-portal → Deployments → 失敗したデプロイをクリック

# 2. ローカルでビルドを試みる
cd C:\Users\simma\Documents\AI\ai-training-portal
npx astro build

# エラーメッセージを確認して原因を特定する
```

#### よくあるビルドエラーと対処

| エラー | 原因 | 対処 |
|---|---|---|
| `Cannot find module` | パッケージ不足 | `npm install` を実行してから再push |
| `frontmatter parse error` | day-XX.mdのYAML書式エラー | 対象ファイルのfrontmatterを修正 |
| `getCollection` エラー | コンテンツスキーマ不一致 | src/content.config.tsの定義を確認 |

#### 復旧手順

1. エラーを修正する
2. ローカルで `npx astro build` が成功するか確認する
3. git pushする（Vercelが自動デプロイ）
4. Vercelのデプロイログで成功を確認する

### ケース3: ワークログが届かない（L2）

#### 確認手順

```
Step 1: GASウェブアプリのURLが正しいか確認する
  → src/pages/submit.astro の GAS_APP_URL を確認
  → ブラウザでGAS_APP_URLに直接アクセスしてフォームが表示されるか確認

Step 2: GASのデプロイ状態を確認する
  → https://script.google.com にアクセス（sim.mail1031@gmail.com）
  → 対象スクリプトを開く → 「デプロイ」→「デプロイを管理」で状態を確認

Step 3: GASの実行ログを確認する
  → GAS管理画面 → 「実行数」でエラーが出ていないか確認
```

#### 復旧手順

**GASウェブアプリが応答しない場合:**
1. GAS管理画面で「新しいデプロイ」を実行（手順書3.5参照）
2. 新しいデプロイURLをコピー
3. `src/pages/submit.astro` の `GAS_APP_URL` を更新
4. git pushしてVercelに反映

**スプレッドシートへの書き込みが失敗している場合:**
1. `1Q7yPdJRXEdMiJBHZNH-Xb72GzGrzY3mpvHCLncWEw1Y` にアクセス
2. スプレッドシートが削除・移動されていないか確認
3. GASスクリプトのスプレッドシートID設定を確認

---

### ケース4: Basic認証が通らない（L1）

#### 確認手順

1. Bitwardenの「dev/ai-training-portal」で正しいパスワードを確認する
2. Vercel管理画面 → Settings → Environment Variables で `BASIC_AUTH_PASSWORD` を確認する

#### 復旧手順

1. Vercel環境変数でパスワードを再設定する
2. Vercelで再デプロイを実行する（または次のgit pushで自動適用）
3. Bitwardenのパスワードを更新する

---

### ケース5: コンテンツ更新が反映されない（L3）

#### 確認手順

```powershell
# GitHubにpushされているか確認
cd C:\Users\simma\Documents\AI\ai-training-portal
git status
git log --oneline -3

# Vercelのデプロイが成功しているか確認
# → Vercel管理画面でデプロイ状況を確認
```

#### 復旧手順

1. `git status` で未pushの変更がないか確認
2. 未pushの場合: `git add . && git commit -m "修正" && git push`
3. Vercelのデプロイが失敗している場合: ケース2を参照
4. デプロイ成功後1〜2分待ってブラウザのキャッシュをクリアしてアクセス
