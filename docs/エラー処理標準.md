**バージョン**: 1.0.0
**作成日**: 2026-06-13

---

## 1. 基本方針

### 鉄則

```
1. エラーを握り潰さない（except: pass / catch {} は禁止）
2. エラーはログに記録してから判断する
3. 「止まる」か「スキップする」かを事前に設計する
4. 再実行できる状態を保つ
5. 人間が介入できる逃げ道を作る
```

### 悪い例（禁止）

```python
# NG: エラーを無視している
try:
    process_record(record)
except:
    pass  # ← 何が起きたかわからない

# NG: エラー内容を消している
try:
    call_api()
except Exception as e:
    print("エラーが起きました")  # ← eを捨てている
```

### 良い例

```python
# OK: エラーをログに記録してスキップ
try:
    process_record(record)
except ValueError as e:
    logger.warning(f"ID:{record['id']} データ不正のためスキップ: {e}")
    skip_count += 1
except Exception as e:
    logger.error(f"ID:{record['id']} 予期しないエラー: {e}", exc_info=True)
    error_count += 1
```

## 2. エラー種別と対処方針

### 2.1 エラー種別一覧

| エラー種別 | 発生原因 | 対処方針 | ログレベル | 処理継続 | 通知 |
|---|---|---|---|---|---|
| **認証エラー** | APIキー無効・期限切れ | 即時停止 | CRITICAL | × | 要 |
| **権限エラー** | アクセス権なし | 即時停止 | CRITICAL | × | 要 |
| **レートリミット** | API呼び出し超過 | 指数バックオフでリトライ | WARNING | ○ | 任意 |
| **タイムアウト** | ネットワーク遅延 | リトライ3回→停止 | ERROR | × | 要 |
| **ネットワークエラー** | 接続失敗 | リトライ3回→停止 | ERROR | × | 要 |
| **データ不正（1件）** | フォーマット異常 | スキップ | WARNING | ○ | 不要 |
| **空データ** | 0件取得 | スキップ or INFO | INFO | ○ | 不要 |
| **重複データ** | 既に処理済み | スキップ | INFO | ○ | 不要 |
| **ファイル不存在** | 入力ファイルがない | 即時停止 | ERROR | × | 要 |
| **書き込み失敗** | 出力先が書けない | 即時停止 | ERROR | × | 要 |
| **設定ファイル不正** | .envや設定JSON異常 | 即時停止 | CRITICAL | × | 要 |
| **想定外エラー** | 予期しない例外 | ログ記録→停止 | CRITICAL | × | 要 |

## 3. エラー別の実装パターン

### 3.1 認証エラー（即時停止）

```python
try:
    client = APIClient(api_key=os.getenv("API_KEY"))
    client.authenticate()
except AuthenticationError as e:
    logger.critical(f"認証失敗: APIキーを確認してください。エラー: {e}")
    sys.exit(1)  # 即時停止
```

### 3.2 レートリミット（指数バックオフ）

```python
import time

def call_api_with_retry(func, max_retries=3):
    for attempt in range(max_retries):
        try:
            return func()
        except RateLimitError as e:
            wait_sec = 2 ** attempt  # 1秒 → 2秒 → 4秒
            logger.warning(f"レートリミット到達。{wait_sec}秒後にリトライ ({attempt+1}/{max_retries})")
            time.sleep(wait_sec)
    logger.error("レートリミット: リトライ上限に達しました")
    raise
```

### 3.3 タイムアウト・ネットワークエラー（リトライ→停止）

```python
import requests
from requests.exceptions import Timeout, ConnectionError

def fetch_data(url, max_retries=3, timeout=10):
    for attempt in range(max_retries):
        try:
            response = requests.get(url, timeout=timeout)
            response.raise_for_status()
            return response.json()
        except Timeout:
            logger.warning(f"タイムアウト ({attempt+1}/{max_retries}回目)")
        except ConnectionError as e:
            logger.warning(f"接続エラー ({attempt+1}/{max_retries}回目): {e}")
        time.sleep(2 ** attempt)
    logger.error(f"データ取得失敗: {max_retries}回リトライしても接続できませんでした")
    raise ConnectionError("接続失敗")
```

### 3.4 データ不正（スキップして継続）

```python
for record in records:
    try:
        validated = validate_record(record)
        process(validated)
        success_count += 1
    except ValueError as e:
        logger.warning(f"ID:{record.get('id', '不明')} バリデーション失敗: {e} → スキップ")
        skip_count += 1
    except Exception as e:
        logger.error(f"ID:{record.get('id', '不明')} 予期しないエラー: {e}", exc_info=True)
        error_ids.append(record.get('id'))
        error_count += 1
```

### 3.5 ファイル不存在（即時停止）

```python
import os, sys

INPUT_FILE = "data/input/records.csv"

if not os.path.exists(INPUT_FILE):
    logger.critical(f"入力ファイルが見つかりません: {INPUT_FILE}")
    sys.exit(1)
```

### 3.6 設定ファイル不正（即時停止）

```python
required_env_vars = ["API_KEY", "SPREADSHEET_ID", "CHATWORK_TOKEN"]

missing = [v for v in required_env_vars if not os.getenv(v)]
if missing:
    logger.critical(f"必須環境変数が未設定です: {missing}")
    sys.exit(1)
```

### 3.7 想定外エラー（最終キャッチ）

```python
try:
    main()
except SystemExit:
    raise  # sys.exit()はそのまま通す
except Exception as e:
    logger.critical(f"予期しないエラーが発生しました: {e}", exc_info=True)
    notify_chatwork(f"[ERROR] 予期しないエラーで停止しました: {e}")
    sys.exit(1)
```

## 4. 「止める」か「スキップ」かの判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 認証情報が無効 | **止める** | 全件失敗するため続けても意味がない |
| 1件のデータが不正 | **スキップ** | 他のデータは処理できる |
| 書き込み先が使えない | **止める** | 全件の出力が失敗するため |
| 1件の書き込みが失敗 | **スキップして記録** | 他のデータは書き込める可能性がある |
| 失敗率が50%を超えた | **止める** | 何か根本的な問題がある可能性 |
| ネットワークが不安定 | **リトライ→判断** | 一時的な問題の可能性がある |

## 5. エラー通知の条件

以下のいずれかが発生した場合はChatwork / Slack等で通知する。

```
[ ] CRITICAL レベルのエラーが発生した
[ ] 処理が完全に停止した
[ ] 失敗件数が全体の10%を超えた
[ ] 24時間以上処理が実行されていない（Cronが死んでいる）
```

### 通知メッセージのテンプレート

```
[ERROR] {プロジェクト名} 障害発生

発生日時: {YYYY-MM-DD HH:MM}
エラー種別: {種別}
エラー内容: {メッセージ}
影響範囲: {件数}件が未処理
実行ID: {exec_id}

対応が必要です。ログを確認してください。
ログパス: logs/{YYYYMMDD}.log
```

## 6. 失敗データの再実行

### 6.1 失敗IDの記録

```python
failed_ids = []

for record in records:
    try:
        process(record)
    except Exception as e:
        failed_ids.append(record["id"])
        logger.error(f"ID:{record['id']} 失敗: {e}")

# 失敗IDをファイルに保存
if failed_ids:
    with open("data/failed_ids.txt", "w") as f:
        f.write("\n".join(str(id) for id in failed_ids))
    logger.warning(f"失敗ID一覧を保存しました: data/failed_ids.txt")
```

### 6.2 再実行時の読み込み

```python
import argparse

parser = argparse.ArgumentParser()
parser.add_argument("--retry-ids", help="再実行するIDをカンマ区切りで指定")
args = parser.parse_args()

if args.retry_ids:
    target_ids = args.retry_ids.split(",")
    records = [r for r in all_records if r["id"] in target_ids]
    logger.info(f"再実行モード: 対象ID {target_ids}")
```

## 7. dry-runでのエラー確認

dry-runでも、バリデーション・API接続確認・入力チェックは本番と同じく実行する。  
dry-runで見つかったエラーは本番でも発生する。

```python
if dry_run:
    logger.info(f"[DRY-RUN] この件は本番では書き込みます: {record['id']}")
else:
    writer.write(record)
```

## 8. エラー処理チェックリスト（実装完了前に確認）

```
[ ] except: pass / catch {} がコード内に存在しない
[ ] 認証エラーで即時停止している
[ ] ネットワークエラーにリトライ処理がある
[ ] リトライ上限に達したら停止している
[ ] エラーのログにIDと理由が含まれている
[ ] exc_info=True でスタックトレースを記録している
[ ] 失敗IDをファイルに保存している
[ ] 失敗率が高い場合に停止する条件がある
[ ] 想定外エラーを最終キャッチしている
[ ] エラー発生時の通知が設定されている
```
