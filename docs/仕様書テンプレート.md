## 1. システム概要

### 1.1 システム名
AI研修ポータル

### 1.2 一言説明
社内ChatGPT活用研修の10日間カリキュラムを提供するWebサイト。ワークログ提出フロー付き。

### 1.3 主要機能

| # | 機能名 | 説明 |
|---|---|---|
| 1 | コンテンツ表示 | Day 1〜10の研修コンテンツをMarkdownから静的ページとして生成・表示 |
| 2 | カリキュラム一覧 | 全10Dayの概要・レベル・所要時間を一覧表示 |
| 3 | GPTs活用ガイド | OpenAI Custom GPTs「AI活用トレーナー」の使い方を案内 |
| 4 | ワークログ提出 | GPTsのまとめをGASウェブアプリに送信してスプレッドシートに蓄積 |
| 5 | Basic認証 | トップページ以外のページに認証を設定（社内向け） |

### 1.4 使用技術・言語

| 項目 | 詳細 |
|---|---|
| フレームワーク | Astro v5（静的サイト生成） |
| CSS | Tailwind CSS v4 + @tailwindcss/typography |
| コンテンツ管理 | Astro Content Collections（Markdown） |
| デプロイ先 | Vercel（Hobbyプラン）|
| バックエンド | Google Apps Script（ウェブアプリ）|
| ワークログ保存先 | Googleスプレッドシート |
| AI連携 | OpenAI Custom GPTs（AI活用トレーナー） |
| バージョン管理 | GitHub（mon-master/ai-training-portal, masterブランチ） |

## 2. 処理フロー

### 2.1 コンテンツ更新フロー

```
管理者がday-XX.mdを編集
  ↓
git push origin master
  ↓
Vercel Webhookが受信（自動）
  ↓
Astroビルド（約1〜2分）
  ↓
Vercel CDNへデプロイ
  ↓
https://ai-training.smr-lab.com に即時反映
```

### 2.2 ワークログ提出フロー

```
参加者がCustom GPTs「AI活用トレーナー」でワーク実施
  ↓
「終了」と入力 → GPTsがまとめを自動生成
  ↓
参加者がまとめを確認（個人情報・機密情報チェック）
  ↓
/submit ページのフォームリンクをクリック
  ↓
GASウェブアプリ（フォームUI）に遷移
  ↓
送信 → GAS doPost処理
  ↓
Googleスプレッドシートに追記
```

## 6. コンテンツ仕様（Markdown frontmatter）

各 `day-XX.md` ファイルのfrontmatter定義：

```yaml
---
title: "Day タイトル"       # 表示タイトル
day: 1                     # Day番号（1〜10）
level: "レベル1〜2"         # ChatGPT活用レベル
duration: "90〜120分"       # 所要時間
goals:                     # 今日のゴール（配列）
  - "ゴール1"
  - "ゴール2"
deliverable: "提出物の説明" # そのDayの成果物
---
```

## 9. エラー処理仕様

| エラー種別 | 対処方法 | 通知 |
|---|---|---|
| Vercelビルド失敗 | 前のビルドが継続稼働。Vercelのメール通知で検知 | Vercelメール |
| GAS応答なし | submit.astroでエラーページを表示（未実装→要対応） | 手動確認 |
| GitHub push失敗 | ローカルのエラーメッセージを確認して解消 | なし |
