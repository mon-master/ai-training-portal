## 2. 処理フロー

### 2.1 全体フロー

```
起動
  ↓
設定ファイル読み込み（.env）
  ↓
ロックファイル確認（二重起動防止）
  ↓
入力データ取得
  ↓
データ検証
  ↓
dry-runチェック（--dry-run フラグ確認）
  ↓
メイン処理
  ├─ 正常：出力・ログ記録
  └─ 異常：スキップ or 停止
  ↓
処理結果サマリーログ出力
  ↓
ロックファイル削除
  ↓
終了
```

|---|---|---|---|
| 1 | 起動・引数解析 | main.py | |

| 3 | ロック確認 | lock.py | |
| 4 | データ取得 | fetcher.py | |

### 7.4 エラーレスポンス

| HTTPステータス | 意味 | 対処 |
|---|---|---|
| 400 | リクエスト不正 | パラメータ確認 |
| 401 | 認証エラー | APIキー確認→即時停止 |
| 403 | 権限エラー | アクセス権確認→即時停止 |
| 429 | レートリミット | 待機してリトライ |
| 500 | サーバーエラー | リトライ→失敗なら停止 |

## 8. ログ仕様

### 8.1 ログファイル

| 項目 | 詳細 |
|---|---|
| 保存先 | `logs/YYYYMMDD.log` |
| 文字コード | UTF-8 |
| ローテーション | 日次 |
| 保存期間 | 30日 |

### 8.2 ログフォーマット

```
[2026-06-13 09:00:00] [INFO] [exec_id:abc123] 処理開始 / 対象件数: 150件
[2026-06-13 09:00:01] [INFO] [exec_id:abc123] 処理完了: 成功148件 / スキップ2件 / 失敗0件
[2026-06-13 09:00:01] [WARNING] [exec_id:abc123] ID:456 - 日付フォーマット不正のためスキップ
[2026-06-13 09:00:01] [ERROR] [exec_id:abc123] API接続失敗: ConnectionTimeout
```

### 8.3 含めない情報

- APIキー・パスワード・トークン
- 個人情報（フルネーム・電話番号・メールアドレス等）

## 9. エラー処理仕様

詳細は `docs/エラー処理標準.md` 参照。

| エラー種別 | 対処方法 | ログレベル | 処理継続 |
|---|---|---|---|
| 認証エラー | 即時停止 | ERROR | × |
| レートリミット | 指数バックオフでリトライ | WARNING | ○ |
| データ不正（1件） | スキップ | WARNING | ○ |
| ネットワークエラー | 3回リトライ→停止 | ERROR | × |
| ファイル不存在 | 即時停止 | ERROR | × |
| 想定外エラー | ログ記録→停止 | CRITICAL | × |

| 項目 | 詳細 |
|---|---|
| 起動方法 | `python main.py --dry-run` |
| 動作 | データ取得・検証まで行い、出力・変更は行わない |
| ログ | `[DRY-RUN]` プレフィックスを付けて出力 |
