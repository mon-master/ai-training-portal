## 1. 概要

### 1.1 このシステムの目的

（例）毎朝9時にネクストエンジンから前日の受注データを取得し、Googleスプレッドシートに自動集計する。

### 1.2 何をするシステムか（一言）

### 1.3 いつ動くか

| 実行タイミング | 頻度 | 設定値 |
|---|---|---|
| 自動実行（Cron） | 毎日 09:00 | `0 9 * * *` |
| 手動実行 | 必要時 | 手順2参照 |

### 1.4 処理の流れ（簡略版）

```
自動起動（Cron）
  → APIからデータ取得
  → スプレッドシートに書き込み
  → ログ記録
  → 完了
```

## 3. 起動方法

### 3.1 手動実行（通常）

```bash
# プロジェクトディレクトリへ移動
cd /home/user/project-name

# 実行
python src/main.py

# dry-runで確認してから実行する場合
python src/main.py --dry-run
python src/main.py  # 問題なければ本番実行
```

### 3.2 特定データのみ再実行

```bash
# 失敗IDを指定して再実行
python src/main.py --retry-ids 123,456,789
```

### 3.3 VPSへのSSHログイン（VPS実行の場合）

```bash
ssh username@xxx.xxx.xxx.xxx -p 22
```

> SSH鍵とパスワードはBitwardenのフォルダ `dev/{project-name}` を確認してください。

## 4. 停止方法

### 4.1 手動で停止する

```bash
# プロセスIDを確認
ps aux | grep main.py

# 停止
kill {プロセスID}
```

### 4.2 緊急停止（すぐに止めたい場合）

```bash
# 強制終了
pkill -f main.py

# ロックファイルが残った場合は削除
rm -f /home/user/project-name/run.lock
```

> ⚠ ロックファイルが残ったまま次回起動すると「実行中です」で起動しません。
> 強制終了後は必ずrun.lockの有無を確認してください。

### 4.3 Cronによる自動実行を一時停止する

```bash
# Cron設定を開く
crontab -e

# 対象行をコメントアウト（先頭に#を付ける）
# 0 9 * * * /home/user/project-name/run.sh

# 保存して終了
```

## 6. 設定変更方法

### 6.1 実行時刻を変更する

```bash
crontab -e
# 「0 9 * * *」の部分を変更する
# 例: 毎日08:30に変更 → 「30 8 * * *」
```

### 6.2 取得対象を変更する（フィルタ条件等）

```bash
vi /home/user/project-name/config/settings.json
# 対象の設定値を変更して保存
```

> settings.jsonの項目説明:
> ```json
> {
>   "filter": {
>     "keyword": "入荷",          ← 収集対象のキーワード
>     "date_range_days": 1        ← 何日分を取得するか
>   }
> }
> ```

### 6.3 APIキーを更新する

```bash
vi /home/user/project-name/.env
# API_KEY=新しいキー に書き換えて保存
```

> ⚠ 変更後はdry-runで動作確認してください。
> 新しいAPIキーは必ずBitwardenにも更新してください。

## 7. ログ確認方法

### 7.1 最新ログを確認する

```bash
# 今日のログ全体
cat /home/user/project-name/logs/$(date +%Y%m%d).log

# 最新50行
tail -n 50 /home/user/project-name/logs/$(date +%Y%m%d).log

# リアルタイムで追いかける（実行中に確認する場合）
tail -f /home/user/project-name/logs/$(date +%Y%m%d).log
```

### 7.2 エラーだけ確認する

```bash
grep "ERROR\|CRITICAL" /home/user/project-name/logs/$(date +%Y%m%d).log
```

### 7.3 処理件数を確認する

```bash
grep "処理完了\|合計" /home/user/project-name/logs/$(date +%Y%m%d).log
```

---

## 8. 成功確認方法

以下の全てが確認できれば正常です。

```
[ ] ログに「処理完了」が出ている
[ ] ログに ERROR / CRITICAL がない
[ ] 処理件数（合計 = 成功 + スキップ + 失敗）が一致している
[ ] 出力先（スプレッドシート等）にデータが正しく書き込まれている
[ ] 失敗件数が0件（またはスキップが仕様の範囲内）
```

---

## 9. 失敗確認方法

### 9.1 異常を検知するサイン

```
- Chatworkに「[ERROR]」の通知が来た
- ログにERROR / CRITICALが出ている
- 出力先（スプレッドシート等）に当日分のデータがない
- Cronが設定時刻を過ぎても実行されていない
```

### 9.2 ログでエラー内容を確認する

```bash
grep "ERROR\|CRITICAL\|WARNING" /home/user/project-name/logs/$(date +%Y%m%d).log
```

→ エラー内容を確認して `docs/障害対応手順書.md` の手順に従って対応してください。

## 10. 手動修正方法

### 10.1 出力データを手動で修正する

1. スプレッドシートを開く
2. 対象行を特定する（実行IDまたは日付で絞り込む）
3. データを直接編集する
4. 修正内容をログファイルに手動でメモを残す（末尾に追記）

### 10.2 バックアップから復元する

```bash
# バックアップ一覧確認
ls /home/user/project-name/backup/

# 復元（例: 2026-06-13の処理前バックアップ）
cp /home/user/project-name/backup/20260613_090000/output.csv \
   /home/user/project-name/data/output.csv
```

## 11. 再実行方法

### 11.1 全件再実行（当日分を最初からやり直す場合）

```bash
# 処理済みIDをクリア
rm -f /home/user/project-name/data/processed_ids.json

# 再実行
python src/main.py
```

> ⚠ 出力先の既存データが上書きされます。バックアップを先に確認してください。

### 11.2 失敗データだけ再実行する

```bash
# 失敗IDファイルを確認
cat /home/user/project-name/data/failed_ids.txt

# 失敗IDを指定して再実行
python src/main.py --retry-ids 123,456,789
```
