## 1. 概要

### 1.1 このシステムの目的

社内AI研修の10日間カリキュラムをWebサイトで提供し、参加者のワークログをGoogleスプレッドシートに自動収集する。

### 1.2 何をするシステムか

- **サイト本体**: https://ai-training.smr-lab.com — 研修コンテンツを提供する静的サイト
- **ワークログ収集**: GASウェブアプリ経由でGoogleスプレッドシートに蓄積

### 1.3 いつ動くか

| 動作タイミング | 内容 |
|---|---|
| 常時 | サイト公開中（Vercel CDN） |
| git push時 | Vercelが自動ビルド・デプロイ（約1〜2分） |
| 参加者がフォーム送信時 | GASがワークログをスプレッドシートに追記 |

## 3. 日常運用手順

### 3.1 研修コンテンツを更新する

```powershell
# 1. プロジェクトフォルダに移動
cd C:\Users\simma\Documents\AI\ai-training-portal

# 2. 最新のコードを取得
git pull

# 3. コンテンツを編集
# src/content/days/day-XX.md を直接編集する

# 4. GitHubにpush（Vercelが自動デプロイ）
git add src/content/days/day-XX.md
git commit -m "day-XX: コンテンツを更新"
git push

# 5. 約1〜2分後にサイトを確認
# https://ai-training.smr-lab.com/days/day-XX
```

### 3.2 コンテンツのファイル構成

```
src/content/days/
├── day-01.md  ← ChatGPTは検索窓ではなく、仕事を手伝う相手
├── day-02.md  ← （タイトル確認）
├── day-03.md  ← 文章作成・整理・確認に使う
├── day-04.md  ← パーソナライズ設定
├── day-05.md  ← プロジェクト機能
├── day-06.md  ← プロンプトを連呼するだけでは使えない理由
├── day-07.md  ← 自分の業務をGPTに渡せる形に整理する
├── day-08.md  ← 実際にGPTを組んでみる
├── day-09.md  ← GPTを業務フローの中に置いてみる
└── day-10.md  ← AIオーケストレーションとして改善提案にまとめる
```

### 3.3 frontmatter（ヘッダー情報）の書き方

```yaml
---
title: "Day タイトル"       # サイトに表示されるタイトル
day: 1                     # Day番号（数字のみ）
level: "レベル1〜2"         # ChatGPT活用レベル（文字列）
duration: "90〜120分"       # 所要時間（文字列）
goals:                     # 今日のゴール（配列）
  - "ゴール1"
  - "ゴール2"
deliverable: "提出物の説明" # そのDayの成果物（文字列）
---
```

## 6. 設定変更手順

### 6.1 Basic認証のパスワードを変更する

1. Vercel管理画面にアクセス（https://vercel.com）
2. ai-training-portal プロジェクトを開く
3. Settings → Environment Variables
4. `BASIC_AUTH_PASSWORD` の値を変更する
5. 再デプロイを実行する（または次のgit pushで自動適用）

### 6.2 GASのデプロイURLを更新する

1. GAS管理画面で新しいデプロイを作成（手順3.5参照）
2. 発行されたURLをコピー
3. `src/pages/submit.astro` の `GAS_APP_URL` を更新する
4. git pushする（Vercelが自動デプロイ）

## 8. よくある質問（FAQ）

| 質問 | 回答 |
|---|---|
| コンテンツを更新したのに反映されない | git pushしたか確認 → Vercelのデプロイ状況を確認（1〜2分待つ） |
