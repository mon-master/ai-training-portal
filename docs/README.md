## このディレクトリについて

`docs/` には、SMR-Labが開発・運用する全ての現場AIツールに共通して使えるテンプレート・チェックリスト・標準書が格納されています。

**新しいプロジェクトを始めるときは、必ずこのディレクトリのドキュメントから始めてください。**

## 推奨ディレクトリ構成

各プロジェクトは以下の構成で作成してください。

```
{project-name}/
├── src/                    # ソースコード
│   ├── main.py
│   ├── config.py
│   ├── fetcher.py
│   ├── processor.py
│   ├── writer.py
│   ├── validator.py
│   └── logger.py
├── config/
│   └── settings.json       # 業務設定（秘密情報以外）
├── logs/                   # 実行ログ（.gitignoreに追加）
├── data/
│   ├── input/
│   └── processed/
├── backup/                 # バックアップ（.gitignoreに追加）
├── tests/
│   ├── test_main.py
│   └── fixtures/
├── docs/                   # このディレクトリのコピー
├── .env                    # 秘密情報（.gitignoreに追加・絶対にコミットしない）
├── .env.example            # .envのひな形（コミットする）
├── .gitignore
├── requirements.txt
└── README.md
```

## 開発フロー（クイックリファレンス）

```
1. AI開発前チェックシート    → 開発GOの判断
        ↓
2. 要件定義テンプレート      → 何を作るかを決める
        ↓
3. 仕様書テンプレート        → どう作るかを決める
        ↓
4. 実装
   ├─ ログ設計標準           → ログの実装
   └─ エラー処理標準         → エラー処理の実装
        ↓
5. セキュリティチェックリスト → push前の確認
        ↓
6. AIコードレビュー依頼      → セルフレビュー
        ↓
7. テスト項目テンプレート     → テスト実施・記録
        ↓
8. 運用手順書テンプレート     → 運用手順書を作成
        ↓
9. 本番投入
        ↓
10. 障害対応手順書テンプレート → 障害時の対応
        ↓
11. 横展開判断シート          → 横展開の判断
```

## Bitwarden への登録ルール

開発で発生したAPIキー・パスワード・トークンは全てBitwardenに登録してください。

```
フォルダ: dev/{project-name}
登録項目:
  - APIキー（サービス名・発行日・有効期限）
  - パスワード
  - スプレッドシートID / スクリプトID
  - 接続情報（URL・ポート等）
```
